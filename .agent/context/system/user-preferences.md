# User Preferences & Global Constraints

## Communication Style
- **Direct:** No preamble, no apologies, no filler
- **Action-Oriented:** Prefer "doing" over "explaining"
- **Structured:** Use headers, lists, tables when helpful
- **Honest:** Label uncertainties as [Unverified]

## Coding Preferences
- **Language:** TypeScript preferred, JavaScript acceptable
- **Style:** Functional, readable, minimal comments
- **Approach:** "Get it working, then clean it up"
- **Testing:** Manual first, automated when mature
- **Architecture:** Simple > Clever

## Technical Environment
- **OS:** Linux (primary development)
- **IDE:** Cursor / VS Code
- **Hardware:** Ryzen 5 7500F, RTX 5070, 32GB RAM
- **Mobile:** Nothing Phone (2a) for testing

## D&D Rules Philosophy
- **RAW First:** Rules as Written takes precedence
- **Edge Cases Matter:** Min-maxers will find them
- **Mathematical Precision:** Calculations must be exact
- **UX Clarity:** Even complex rules should be understandable

## Session Management
- **Prime:** Always start with `/prime-agent [project]`
- **Cleanup:** End sessions with `/cleanup`
- **Abstract:** Extract patterns into reusable commands
- **Logs:** Maintain continuity via work-log

## Anti-Patterns to Avoid
- Over-engineering simple solutions
- Adding features not requested
- Speculating without verification
- Breaking existing patterns for "improvements"
- Long theoretical discussions without action
